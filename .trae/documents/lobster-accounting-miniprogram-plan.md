# 龙虾记账微信小程序 — 开发计划书

## 一、项目概述

### 1.1 背景
龙虾商户在销售过程中，传统手工记账效率低、易出错、难以统计。本项目旨在开发一款微信小程序，让卖家在手机上快速记账，买家通过分享链接查看消费记录，实现数字化记账管理。

### 1.2 核心目标
- 卖家：快速录入交易、管理赊账、查看统计报表
- 买家：通过分享链接查看自己的消费和欠款记录
- MVP 优先：2 周内可测试核心记账功能

### 1.3 需求确认摘要

| 维度 | 决策 |
|------|------|
| 用户角色 | 卖家为主 + 买家通过分享链接查账 |
| 交易字段 | 基础信息 + 付款状态 + 配送信息 |
| 数据存储 | 云端存储（自建后端） |
| 交易规模 | 中等规模（几十商户，日均几百笔） |
| 统计报表 | 收支统计、欠款排行、品种销量分析、报表导出 |
| 买家查账 | 分享链接查看，无需登录 |
| 技术栈 | 前端微信小程序 + 自建后端服务 |
| 上线策略 | 快速上线 MVP |

### 2.2 技术选型

| 层级 | 技术 | 理由 |
|------|------|------|
| 小程序前端 | 微信原生小程序 + TypeScript | 性能最优，API 支持最全，分享能力原生支持 |
| UI 组件库 | Vant Weapp | 成熟稳定，组件丰富，适合后台管理类界面 |
| 后端框架 | Node.js + Koa2 | 轻量灵活，适合中等规模，开发效率高 |
| ORM | Sequelize | Node.js 生态成熟 ORM，支持 MySQL |
| 数据库 | MySQL 8.0 | 关系型数据适合交易记录，事务支持完善 |
| 缓存 | Redis | 会话管理、统计缓存、热点数据加速 |
| 文件存储 | 本地 / OSS | 导出文件临时存储 |
| 部署 | Docker + Nginx | 标准化部署，易扩展 |

## 三、功能模块与分阶段计划

### Phase 1：MVP 核心记账（第 1-2 周）

> 目标：卖家可以录入交易、查看列表，买家可以通过链接查账

#### 1.1 卖家端

**1.1.1 商户注册/登录**
- 微信一键登录（获取 OpenID）
- 首次登录填写店铺名称和联系电话
- 后续登录自动进入工作台

**1.1.2 快速记账**
- 表单页面，字段：
  - 买家姓名（支持从历史买家选择或新建）
  - 买家手机号（选填）
  - 龙虾品种（预设常用 + 自定义输入）
  - 重量/数量 + 单价 → 自动计算总价
  - 付款状态（未付/已付/部分付款）+ 已付金额
  - 配送地址（选填）
  - 配送状态（待配送/配送中/已送达）
  - 预计送达时间（选填）
  - 备注（选填）
- 保存后跳转交易列表

**1.1.3 交易列表**
- 按时间倒序展示所有交易
- 筛选：按付款状态（未付/部分付款/已付）、按买家、按日期范围
- 点击单条可编辑/补录付款

**1.1.4 买家管理**
- 买家列表，显示姓名、手机号、累计消费、当前欠款
- 点击买家查看该买家所有交易记录

#### 1.2 买家查账

**1.2.1 分享链接生成**
- 买家详情页可生成专属小程序分享卡片
- 分享链接携带买家 `share_token` 参数

**1.2.2 买家查看页面**
- 无需登录，通过 `share_token` 直接查看
- 展示：买家姓名、所有交易记录、累计消费、当前欠款
- 仅可查看，不可编辑

#### 1.3 后端 API（MVP）

| API | 方法 | 说明 |
|-----|------|------|
| /api/auth/login | POST | 微信登录 |
| /api/merchants/profile | GET/PUT | 商户信息 |
| /api/buyers | GET/POST | 买家列表/新增 |
| /api/transactions | GET/POST | 交易列表/新增 |
| /api/transactions/:id | GET/PUT | 交易详情/编辑 |
| /api/buyers/:token/records | GET | 买家查账（公开） |

### Phase 2：统计报表与导出（第 3-4 周）

#### 2.1 收支统计
- 按日/周/月维度展示：总收入、已收款、未收款、部分收款
- 折线图展示收入趋势
- 环比/同比对比

#### 2.2 欠款排行
- 按欠款金额倒序排列买家
- 显示：买家姓名、总消费、已付、欠款金额、最近交易时间
- 支持一键催款（生成带欠款信息的微信分享卡片）

#### 2.3 品种销量分析
- 各龙虾品种的销量（斤数）和收入占比
- 饼图 + 柱状图展示
- 按时间段筛选

#### 2.4 报表导出
- 导出为 Excel 文件
- 支持导出维度：交易明细表、收款汇总表、欠款明细表
- 通过微信聊天发送文件或复制下载链接

#### 2.5 后端 API（Phase 2）

| API | 方法 | 说明 |
|-----|------|------|
| /api/stats/overview | GET | 收支概览 |
| /api/stats/trend | GET | 收入趋势 |
| /api/stats/debt-ranking | GET | 欠款排行 |
| /api/stats/product-analysis | GET | 品种分析 |
| /api/export/excel | POST | 导出 Excel |

### Phase 3：增强功能（第 5-6 周）

#### 3.1 付款记录
- 一笔交易支持多次付款记录
- 记录付款方式和时间
- 自动更新交易付款状态

#### 3.2 配送状态管理
- 更新配送状态，支持状态流转
- 配送中自动通知买家（通过微信服务通知或订阅消息）

#### 3.3 数据安全
- `share_token` 定期更换机制
- 商户操作日志
- 数据备份策略

#### 3.4 体验优化
- 记账页面常用字段记忆（上次选择的品种、单价）
- 语音输入备注（调用微信语音识别）
- 快捷记账：从买家详情页直接新增该买家的交易

## 七、风险与注意事项

1. **微信小程序审核**：记账类小程序需注意审核规范，避免涉及金融相关敏感词，定位为"交易记录管理工具"
2. **数据安全**：`share_token` 需足够随机且不易猜测，建议使用 UUID v4 + 时间戳哈希
3. **并发控制**：中等规模下 MySQL 单库可承受，但需注意交易写入时的乐观锁/悲观锁
4. **微信登录依赖**：后端需配置正确的 AppID 和 AppSecret，开发阶段可用测试号
5. **Excel 导出**：小程序无法直接下载文件，需通过微信聊天发送或生成临时下载链接
6. **买家隐私**：买家查账页面仅展示交易记录，不暴露其他买家信息
